# Dunbar Link Beta 브라우저/세션 초기화 안내

- 작성일: 2026-06-11
- 용도: 운영 문서. 기존 테스트폰의 쿠키/로그인 세션 때문에 초대/수락/로그인 흐름이 꼬일 수 있다. 이 문서는 **앱 문제가 아닌 브라우저 세션 문제를 먼저 분리**하기 위한 운영 기준이다.
- 함께 보기: [beta-tester-recruit-kit.md](./beta-tester-recruit-kit.md) (모집/안내 문구) · [beta-feedback-log-template.md](./beta-feedback-log-template.md) (피드백 기록) · [beta-backlog-triage.md](./beta-backlog-triage.md) (분류 기준)

## 1. 일반 테스터에게 우선 안내할 방법 (순서대로)

1. **Chrome 시크릿 탭**에서 초대 링크 열기
2. 카톡 안에서 바로 열지 말고 **외부 브라우저**로 열기
3. 기존에 열었던 브라우저 말고 **다른 브라우저**로 열기
4. 그래도 안 되면 **dunbar-link.vercel.app 사이트 데이터만 삭제**
   (Chrome → 사이트 설정 → dunbar-link.vercel.app → 데이터 삭제. 전체 브라우저 데이터 삭제가 아님)

## 2. 일반 테스터 안내 문구 (복사용)

```
테스트가 꼬이면 아래 중 하나로 다시 열어줘 😊

1. Chrome 시크릿 탭에서 초대 링크 열기
2. 카톡 안에서 바로 열지 말고 외부 브라우저로 열기
3. 기존에 열었던 브라우저 말고 다른 브라우저로 열기
4. 그래도 안 되면 dunbar-link.vercel.app 사이트 데이터 삭제
```

## 3. 운영자 판단 기준 (세션 꼬임으로 의심할 정황)

아래 정황이면 앱 버그로 분류하기 전에 위 1번 순서로 먼저 재확인한다.

- 초대 링크가 이상한 화면으로 열림
- 이전 계정/이전 테스트 상태가 남아 있음
- 수락 화면 대신 기존 대시보드로 이동함
- 카톡 인앱 브라우저 또는 Edge/Whale/Chrome 세션이 섞인 정황이 있음

## 4. 하지 말아야 할 것

- 일반 테스터에게 **ADB를 요구하지 않는다**.
- 휴대폰 **전체 브라우저 데이터 삭제를 먼저 요구하지 않는다** (사이트 데이터 삭제로 충분).
- **KakaoTalk 앱 데이터 삭제를 요구하지 않는다** (대화/로그인이 날아갈 수 있다).
- 로그인/쿠키/캐시 삭제가 어떤 영향을 주는지 설명하지 않고 앱 데이터 삭제를 안내하지 않는다.

## 5. ADB reset helper 관련 (내부 전용)

- ADB 방식은 **내부 운영자/개발자용**이다. 일반 테스터에게 배포하지 않는다.
- USB 디버깅이 필요하다.
- Chrome/Edge/Whale/Samsung Internet **앱 데이터 삭제는 해당 브라우저의 로그인/쿠키/캐시를 모두 지운다** — 테스터 개인 계정에 영향이 크다.
- **KakaoTalk 앱 데이터는 삭제 대상에서 제외**해야 한다.
- 내부 재현 테스트에서 깨끗한 세션이 꼭 필요할 때만 사용한다.

## 6. 현재 권장 결론

- 일반 테스터: **시크릿 탭 → 외부 브라우저 → 다른 브라우저 → 사이트 데이터 삭제** 순서로만 안내한다.
- ADB 초기화: 내부 재현 테스트가 꼭 필요할 때만, 운영자 본인 기기에서 사용한다.
