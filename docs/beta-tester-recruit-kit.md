# Dunbar Link 베타 테스터 모집 문구

- 작성일: 2026-06-10
- 용도: 친구/지인에게 바로 보낼 수 있는 카톡/문자 테스트 요청 문구 모음.
- 톤 원칙: "버그 찾아줘"가 아니라 **"한번 써보고 막히는 부분 알려줘"**. 5분 안에 끝나는 느낌으로.
- 함께 보기: [beta-test-guide.md](./beta-test-guide.md) (테스트 절차 상세) · [beta-test-operator-checklist.md](./beta-test-operator-checklist.md) (운영자 체크리스트) · 피드백 기록은 [beta-feedback-log-template.md](./beta-feedback-log-template.md)

---

## 1. 짧은 카톡 문구 (기본)

```
안녕 😊
내가 만들고 있는 관계 관리 앱 베타 테스트 중인데,
5분 정도만 써보고 막히는 부분 알려줄 수 있어?

테스트는 간단해.

1. 링크 열기
2. 이름 입력하고 시작
3. 홈 화면에서 사람 위치 확인
4. 신호 하나 보내기
5. 받은 신호에서 답신호 한번 보내기

써보다가 이상한 화면, 헷갈리는 문구, 안 되는 버튼 있으면
캡처해서 보내주면 진짜 도움 돼 ㅎㅎ

링크: [여기에 링크 붙여넣기]
```

## 2. 더 친한 사람용 문구

```
야 ㅎㅎ
내가 만든 앱 거의 베타까지 왔는데
한번만 테스트해줘.

5분이면 되고, 어려운 거 아니야.

링크 열고
이름 넣고
신호 보내보고
답신호 되는지만 보면 돼.

하다가 이상하거나 헷갈리는 거 있으면 캡처 하나만 보내줘.
그게 제일 도움 된다 😊

링크: [여기에 링크 붙여넣기]
```

## 3. 정중한 지인용 문구

```
안녕하세요.
제가 개발 중인 관계 관리 앱을 베타 테스트하고 있습니다.

혹시 5분 정도 사용해보시고
막히는 부분이나 헷갈리는 부분을 알려주실 수 있을까요?

확인해주시면 좋은 부분은 아래 정도입니다.

- 초대 링크가 잘 열리는지
- 이름 입력 후 시작이 되는지
- 홈 화면이 자연스러운지
- 신호 보내기 / 답신호가 되는지
- 모바일 화면에서 잘리는 부분이 없는지

사용 중 이상한 부분이 있으면 캡처로 보내주시면 큰 도움이 됩니다.

링크: [여기에 링크 붙여넣기]
```

## 4. 테스터에게 부탁할 체크 항목

테스트 후 가볍게 물어볼 체크리스트 (전부 물어볼 필요 없음, 2~3개면 충분):

- 링크가 잘 열렸나요?
- 이름 입력이 쉬웠나요?
- 홈 화면이 이해됐나요?
- 사람 위치를 바꾸는 게 가능했나요?
- 신호 보내기가 됐나요?
- 답신호가 됐나요?
- 화면이 잘리거나 버튼이 안 눌린 곳이 있었나요?
- 헷갈린 문구가 있었나요?
- 다시 쓰고 싶은 느낌이 있었나요?

## 5. 캡처 요청 문구

문제가 생기면 아래 3가지만 보내달라고 안내한다:

```
혹시 막히면 이 3개만 보내줘!

1. 어떤 휴대폰인지 (예: 갤럭시 S24, 아이폰 15)
2. 어떤 화면에서 막혔는지
3. 캡처 1장
```

## 6. 운영자 메모

테스터에게 너무 많은 걸 요구하지 말 것. 받은 피드백은 [beta-feedback-log-template.md](./beta-feedback-log-template.md)의 표에 기록하고, 분류는 [beta-backlog-triage.md](./beta-backlog-triage.md) 기준을 따른다.

좋은 질문:

- 어디서 막혔어?
- 뭐가 헷갈렸어?
- 누르기 어려운 버튼 있었어?
- 다시 쓰고 싶은 느낌이 있었어?

나쁜 질문:

- UX 전체 평가해줘
- 관계 그래프 개념 이해돼?
- 기능 뭐 추가하면 좋을까?
- 시장성이 있어 보여?
