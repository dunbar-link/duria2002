# P4 — 2초 음성 신호 MVP 실사용 관찰 계획

- 성격: 운영/제품 판단 문서. **이 기간 동안 새 기능 구현 금지.** 코드 변경 없음.

---

## 1. 현재 제품 한 줄

- 카톡은 대화, 던바링크는 2초 신호다.
- 지금 검증 대상은 **2초 음성 신호** 하나다.
- 영상은 보류다(docs/p4-2-video-signal-design-audit.md 참고 — 음성 사용성 확인 전까지 구현 금지).

## 2. 관찰 기간

- 기본 **7일**.
- 신호가 부족하거나(전송 건수가 너무 적어 판단 불가) 애매하면 **14일**까지 연장.
- 이 기간 동안 8절의 "지금 하지 말 것" 항목을 지킨다 — 새 기능/개편 없이 있는 그대로 써 본다.

## 3. 핵심 관찰 질문

1. 사람들이 2초 음성을 실제로 보내고 싶어 하는가(억지로 눌러보는 게 아니라)?
2. 받은 사람은 부담 없이 듣는가, 아니면 "이걸 지금 들어야 하나" 망설이는가?
3. 신호함/빨간점이 다시 앱을 열게 만드는가?
4. 24시간 후 사라지는 정책이 자연스러운가, 아니면 "왜 벌써 없어졌지" 혼란을 주는가?
5. 한 번 듣고 사라지는 듯한 UX가 좋은가, 혼란스러운가(실제로는 만료 전까지 재생 가능하지만 "짧고 휘발적"이라는 인상 자체를 다루는 질문)?
6. 이모지와 음성 중 무엇이 더 자주 쓰이는가?
7. 영상 버튼 "준비중" 표시가 기대를 만드는가(자꾸 눌러보게 되는가), 아니면 그냥 무시되는가?

## 4. 최소 수동 기록 항목

전송/수신 한 건마다 아래 표에 한 줄씩 기록한다. 대장 본인 기기(폰 카카오 계정 / PC 이메일 OTP 계정) 간 왕복부터 시작.

| 날짜 | 보낸 사람 | 받은 사람 | 신호 종류(emoji/voice) | 전송 성공 여부 | 실제 재생했는지 | 빨간점 확인 여부 | push 수신 여부 | 재방문 여부 | 불편/혼란 메모 |
|---|---|---|---|---|---|---|---|---|---|
| | | | | | | | | | |

- "전송 성공 여부": 신호함에 실제로 기록됐는지(에러 없이 끝까지 갔는지).
- "재방문 여부": 이 신호를 계기로 앱을 다시 열었는지(신호함/빨간점을 보고 왔는지, 아니면 다른 이유로 열었는지).
- "불편/혼란 메모": 짧게라도 그 자리에서 든 생각을 남긴다(나중에 재구성하면 왜곡되기 쉬움).

## 5. 성공 기준

- 음성 신호 전송이 기술적으로 안정적이다(에러/실패 없이 매번 됨).
- 받은 사람이 재생까지 자연스럽게 완료한다(중간에 포기하거나 헷갈리지 않음).
- 빨간점/신호함으로 재방문이 실제로 발생한다.
- push는 모바일 기준 정상 수신된다(PC는 브라우저 권한 이슈로 별도 취급, 6절 참고).
- 2초 제한이 불편보다 장점으로 느껴진다("짧아서 편하다"는 반응).
- 영상 없이도 관계 신호로서의 가치가 느껴진다.

## 6. 실패/보류 기준

- 음성 보내기가 어색해서 거의 안 쓰게 된다.
- 받은 사람이 재생하지 않는다(신호함에만 쌓이고 안 들음).
- 24시간 만료가 혼란을 준다("어? 없어졌네" 부정적 반응이 반복됨).
- 신호함/빨간점이 의미 있게 재방문을 만들지 못한다.
- push 없이는(또는 push가 와도) 앱 재방문이 거의 안 된다.
- 영상에 대한 요구가 실제로는 나오지 않는다(즉 영상 보류가 아까울 게 없다는 뜻이기도 함 — 이 경우는 오히려 A안 유지의 근거가 된다).
