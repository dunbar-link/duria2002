## 1. 기록 원칙

- 피드백 1건은 바로 개발하지 않는다.
- 같은 문제가 2회 이상 반복되면 P2 후보.
- 핵심 루프(초대/수락/홈 조작/신호)를 막으면 P1.
- 데이터/상대 정보 오류는 P0/P1.
- 새 기능 제안은 Backlog 또는 Not Now로 둔다.

| 날짜 | 테스터 | 기기/브라우저 | 상황 | 피드백 원문 | 분류 | 재현 여부 | 조치 | 상태 |
|---|---|---|---|---|---|---|---|---|
| 2026-06-__ | A | iPhone / Safari | 초대 수락 | 이름 넣고 시작은 됐는데 홈이 뭔지 모르겠음 | P2 | 미확인 | 문구 개선 후보 | 메모 |
| 2026-06-__ | B | Android / Chrome | 신호함 | 답신호는 됐는데 버튼이 작아 보임 | P2 | 미확인 | 반복 시 검토 | 메모 |

## 3. 분류 기준 요약

상세 기준은 [beta-backlog-triage.md](./beta-backlog-triage.md) 참조. 기록 시 빠른 판단용 요약:

**P0**
- 앱 사용 불가
- 초대/수락 불가
- 데이터 손실
- 잘못된 사람에게 신호 전송
- 개인정보/상대 정보 오표시

**P1**
- 핵심 루프 강하게 차단
- 모바일 주요 조작 불가
- 이름/사진 sync 불일치 재발
- 삭제 연결/알 수 없음 재발

**P2**
- 사용은 가능하지만 불편
- 버튼 위치/문구/가시성
- ghost 비주얼
- 스크롤/드래그 체감

**Backlog**
- 이모지 확장
- 답신호 고도화
- 폴더 UX
- 추천/탐색 고도화

**Not Now**
- 채팅앱화
- 결제 대개편
- DB schema 대개편
- 추천 알고리즘 대개편
- 대규모 graph expansion

## 4. 다음 개발 착수 조건

- P0: 즉시 중단 후 수정
- P1: 1회라도 재현되면 우선 수정
- P2: 2회 이상 반복되면 수정 후보
- Backlog: 베타 안정성 통과 후 별도 승인
- Not Now: 명시 승인 전 금지
