## 4. 동결 기간 금지 작업 (30일)

아래는 동결 기간 중 **어떤 지시가 와도 구현하지 않는다.** 새 기능 제안이 들어오면 이 문서를 근거로 재개 조건(§5)부터 확인.

- 영상 신호 구현
- 거리 스코어 (관계 거리 계산)
- 코인 시스템
- 유명인 단계 계산
- 소개받기 기능
- 인맥 탐색
- 친구 검색
- 추천 알고리즘
- 새로운 보상 시스템
- UI 대개편

허용되는 작업(동결 대상 아님):
- 보안 패치, 심각한 버그 수정 (사용자 데이터 손실·인증 우회 등급)
- 이미 완료된 관찰 계획(P4 음성 MVP observation plan)에 따른 관찰 결과 기록
- 기존 코드의 라벨/타입 정합성 등 유지보수성 개선 (이번 동결 문서 작성 자체가 이 범주)

## 5. 재개 조건

아래 중 **하나라도** 충족되면 재개를 검토한다 (자동 재개 아님 — 대장이 판단):

1. 실제 사용자 **5명 이상**이 **7일 동안 반복 사용**
2. 명확한 **유료 고객 / 기관 / 팀 사용처** 발견
3. 다른 프로젝트에서 **관계 엔진 재사용 필요** 발생
4. 대장이 직접 **테스트 자원 2명 이상** 확보

재개 시 진행 순서: 위 조건 중 어떤 것이 충족됐는지 먼저 확인 → 해당 조건에 맞는 최소 범위로 재개 (예: 재사용 필요 발생 시 해당 프로젝트에 필요한 최소 기능만 우선).
