# AGENT_RULES.md — Dunbar Link AI 개발 운영 규칙

이 파일은 Claude Code가 Dunbar Link 프로젝트에서 따라야 할 개발 원칙을 정의합니다.
모든 작업의 기준 문서로 사용됩니다.

---

## 프로젝트 정보

- **프로젝트명:** Dunbar Link
- **목적:** 신호 기반 관계 유지 앱
- **UX 방향:** 모바일 퍼스트, 최소 텍스트, 아이콘 중심
- **앱 성격:** 채팅앱이 아닌 관계 유지 앱

---

## 핵심 제품 철학

- 사람 중심 그래프 — 사용자 간 연결이 핵심
- 기관/학교/회사 노드는 사용자 그래프에 노출하지 않음
- 연락보다 "신호(Signal)" 중심
- 행동 유도 우선 — 설명보다 즉시 액션

---

## 개발 원칙

1. **조사 → 계획 → 승인 → 수정 → 검증** 순서를 반드시 지킨다
2. 무단 리팩토링 금지 — 요청 범위 밖의 변경 절대 금지
3. 수정 범위 최소화 — 한 번에 **1~3개 파일**만 수정 권장
4. 삭제보다 **deprecated 처리** 우선
5. 가능한 한 파일 단위로 안정적으로 수정

---

## UI 원칙

- 모바일 중심 레이아웃
- 텍스트 최소화, 아이콘/직관 중심
- 홈 bottom nav 고정 유지
- 헤더는 스크롤로 사라지지 않도록 유지 (`sticky top-0`)
- 그래픽 우선, 긴 설명 금지

---

## 코드 구조 원칙

| 위치 | 역할 |
|------|------|
| `*-types.ts` / `types/` | 타입 정의만 |
| `*-utils.ts` / `utils/` | 순수 함수만 (부수효과 없음) |
| `hooks/` / `use-*.ts` | React 훅만 |
| `*.tsx` 컴포넌트 | JSX/UI 렌더링만 |

- 역할 섞기 금지 — 컴포넌트 안에 비즈니스 로직 직접 작성 지양
- 신규 파일 생성 전 기존 파일 재사용 가능 여부 먼저 확인

---

## 위험 작업 금지

- `.env.local` 내용 출력 금지
- Supabase 키 (ANON_KEY, Service Role Key 등) 출력 금지
- Service Role Key 클라이언트 노출 금지
- 대량 파일 삭제 금지
- `git push` 금지
- `git commit` 금지 (명시적 요청 없으면)

---

## Supabase 원칙

- 사람 / 신호 / 초대 데이터 안정성 최우선
- 기존 `invite` / `signals` 흐름 깨지지 않도록 유지
- 기존 `localStorage` 키 유지 우선 — 무단 변경 금지
- Admin 클라이언트(`lib/supabase/admin.ts`)는 서버 전용, 클라이언트 노출 절대 금지
